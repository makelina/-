# Тест-кейсы: Оплата заказа банковской картой

---

### ПОЗ-01
**ID:**  
ПОЗ-01

**Название:**  
Оплата заказа банковской картой через форму оплаты при успешном прохождении 3-D Secure

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в статусе «Ожидает оплаты»
4. Есть валидная тестовая карта с достаточным балансом
5. Открыта страница оформления заказа с формой оплаты

**Шаги:**  
1. Ввести валидные данные карты в форму оплаты
   - В поле «Номер карты» ввести: 4242 4242 4242 4242
   - В поле «Срок действия» ввести: 12/25
   - В поле «CVV» ввести: 123
   - В поле «Имя держателя» ввести: IVAN IVANOV
2. Нажать кнопку «Оплатить»
   - Кликнуть по кнопке «Оплатить» в форме оплаты
   - Дождаться загрузки страницы платёжного шлюза
3. Пройти аутентификацию 3-D Secure
   - На странице банка ввести код из SMS: 123456
   - Нажать кнопку «Подтвердить»
4. Дождаться возврата на сайт магазина
   - Автоматический редирект на страницу результата оплаты
   - Ожидание полной загрузки страницы

**Ожидаемый результат:**  
Система отображает страницу «Оплата успешна», меняет статус заказа на «Оплачен», отправляет чек на email и подтверждает списание средств.

НЕГ-01

**Название:**

2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в статусе «Ожидает оплаты»
4. Есть тестовая карта с недостаточным балансом
5. Открыта страница оформления заказа с формой оплаты

**Шаги:**  
1. Ввести данные карты с недостаточным балансом
   - В поле «Номер карты» ввести: 4000 0000 0000 0002
   - В поле «Срок действия» ввести: 12/25
   - В поле «CVV» ввести: 123
   - В поле «Имя держателя» ввести: IVAN IVANOV
2. Нажать кнопку «Оплатить»

2. Пользователь авторизован в личном кабинете

4. Симулирована недоступность платёжного шлюза (на стороне тестирования)
5. Открыта страница оформления заказа с формой оплаты

**Шаги:**  
1. Ввести валидные данные карты
   - В поле «Номер карты» ввести: 4242 4242 4242 4242
   - В поле «Срок действия» ввести: 12/25
   - В поле «CVV» ввести: 123

### ВАЛ-01
**ID:**  
ВАЛ-01

**Название:**  
Валидация обязательных полей в форме оплаты банковской картой при попытке отправки формы с пустыми значениями

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в корзине
4. Открыта страница оформления заказа с формой оплаты банковской картой

**Шаги:**  
1. Оставить все поля формы пустыми
   - Поле «Номер карты» — не заполнять
   - Поле «Срок действия» — не заполнять
   - Поле «CVV» — не заполнять
   - Поле «Имя держателя» — не заполнять
2. Нажать кнопку «Оплатить»
   - Кликнуть по кнопке «Оплатить» в форме оплаты
   - Зафиксировать реакцию интерфейса

**Ожидаемый результат:**  
Система блокирует отправку формы, отображает сообщения «Поле обязательно для заполнения» и не отправляет запрос на сервер.

### ВАЛ-02
**ID:**  
ВАЛ-02

**Название:**  
Валидация формата номера карты и CVV-кода в форме оплаты при вводе некорректных символов

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в корзине
4. Открыта страница оформления заказа с формой оплаты

**Шаги:**  
1. Ввести невалидное значение в поле «Номер карты»
   - Ввести буквы: ABCD EFGH IJKL MNOP
   - ИЛИ ввести 12 цифр: 4242 4242 4242
2. Ввести невалидное значение в поле «CVV»
   - Ввести буквы: ABC
   - ИЛИ ввести 4 цифры: 1234
3. Нажать кнопку «Оплатить»
   - Кликнуть по кнопке «Оплатить» в форме оплаты
   - Зафиксировать реакцию интерфейса

**Ожидаемый результат:**  
Система блокирует отправку формы, отображает сообщения о неверном формате данных и не отправляет запрос к платёжному шлюзу.

### ИНТ-01
**ID:**  
ИНТ-01

**Название:**  
Отмена оплаты на странице 3-D Secure банка при переходе из формы оплаты веб-приложения

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в статусе «Ожидает оплаты»
4. Пользователь инициировал оплату и перешёл на страницу 3-D Secure банка
5. Открыта страница аутентификации банка

**Шаги:**  
1. На странице 3-D Secure нажать кнопку «Отмена»
   - Найти кнопку «Отмена» или «Cancel» на странице банка
   - Кликнуть по кнопке «Отмена»
   - ИЛИ закрыть вкладку браузера с страницей 3-D Secure
2. Вернуться на вкладку магазина
   - Переключиться на исходную вкладку веб-приложения
   - Зафиксировать отображаемую страницу

**Ожидаемый результат:**  
Система возвращает пользователя на страницу заказа, отображает статус «Оплата отменена», не меняет статус заказа в БД и не списывает средства.

### ИНТ-02
**ID:**  
ИНТ-02

**Название:**  
Обновление статуса заказа в базе данных при получении успешного ответа от платёжного шлюза

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Заказ создан и находится в статусе «New»
4. Подготовлен мокированный успешный ответ от платёжного шлюза
5. Есть доступ к панели администратора для проверки БД

**Шаги:**  
1. Провести оплату с успешным ответом от шлюза
   - Ввести валидные данные карты в форму оплаты
   - Нажать кнопку «Оплатить»
   - Получить мокированный успешный ответ от шлюза
2. Проверить запись о заказе в базе данных администратора
   - Войти в панель администратора
   - Найти заказ по номеру
   - Зафиксировать текущий статус заказа
3. Проверить историю изменений статуса заказа
   - Открыть историю изменений заказа
   - Зафиксировать все изменения статуса с временными метками

**Ожидаемый результат:**  
Система меняет статус заказа на «Paid», записывает transaction_id в историю и фиксирует время обновления статуса.

### ИНТ-03
**ID:**  
ИНТ-03

**Название:**  
Отправка email-чека пользователю при успешной оплате заказа через платёжный шлюз

**Предусловия:**  
1. Сайт: https://example-shop.com (веб-приложение интернет-магазина)
2. Пользователь авторизован в личном кабинете
3. Email пользователя верифицирован
4. Проведены две транзакции: одна успешная, одна отклонённая
5. Есть доступ к почтовому ящику пользователя для проверки

**Шаги:**  
1. Выполнить сценарий ПОЗ-01 (Успех)
   - Провести успешную оплату по шагам из ПОЗ-01
   - Зафиксировать время отправки транзакции
2. Выполнить сценарий НЕГ-01 (Отказ)
   - Провести оплату с отклонением по шагам из НЕГ-01
   - Зафиксировать время отправки транзакции
3. Проверить почтовый ящик пользователя
   - Войти в email-ящик пользователя
   - Проверить входящие письма за последний час
   - Зафиксировать количество и содержание писем с чеками

**Ожидаемый результат:**  
Система отправляет чек на email только после успешной оплаты, не отправляет чек после отказа и заполняет в чеке корректные данные заказа.
